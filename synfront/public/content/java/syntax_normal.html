Basic Syntax and Structure

Java programs follow a specific structure and syntax. Understanding these fundamentals is essential for writing correct code.

Program Structure

A basic Java program consists of:

// This is a simple Java program

public class HelloWorld {
    public static void main(String[] args) {
        // Your code goes here
        System.out.println("Hello, World!");
    }
}

Let's break down the components:

  • public class HelloWorld: Defines a class named HelloWorld. The class name must match the filename (HelloWorld.java).
  • public static void main(String[] args): The main method, which is the entry point of the program. The Java Virtual Machine (JVM) calls this method when the program starts.
  • System.out.println("Hello, World!");: Outputs text to the console.

Comments

Comments are used to explain code and are ignored by the compiler:

// This is a single-line comment

/* This is a
   multi-line comment */

/**
 * This is a documentation comment (Javadoc)
 * Used to generate documentation for your code
 */

Statements and Semicolons

In Java, statements are terminated with semicolons:

int x = 5;  // Declaration statement
System.out.println(x);  // Output statement
x = x + 1;  // Assignment statement

Blocks of Code

Blocks are defined using curly braces and group multiple statements:

if (x > 0) {
    System.out.println("Positive");
    x = 0;
}

Identifiers

Identifiers are names used for classes, variables, and methods:

  • Must begin with a letter, underscore (_), or dollar sign ($)
  • Subsequent characters can be letters, digits, underscores, or dollar signs
  • Cannot use Java reserved keywords
  • Case-sensitive (age, Age, and AGE are different identifiers)

Naming Conventions

Java follows specific naming conventions:

  • Classes: Start with uppercase letter, use CamelCase (e.g., HelloWorld)
  • Methods and variables: Start with lowercase letter, use camelCase (e.g., firstName)
  • Constants: All uppercase with underscores (e.g., MAX_VALUE)
  • Packages: All lowercase, use reverse domain name convention (e.g., com.example.project)

Package Declaration

Packages are used to organize classes and avoid naming conflicts:

package com.example.myproject;

public class MyClass {
    // Class content
}

Import Statements

Import statements allow you to use classes from other packages:

import java.util.ArrayList;
import java.util.Scanner;
// Or import all classes from a package
import java.util.*;

Key points to remember:

  • Java is case-sensitive
  • Class names should match file names
  • Every Java application must have a main method
  • Statements end with semicolons
  • Code blocks are enclosed in curly braces {}
  • Follow Java naming conventions for readability
  • Use packages to organize your code
  • Import statements should be placed after the package declaration and before the class declaration
