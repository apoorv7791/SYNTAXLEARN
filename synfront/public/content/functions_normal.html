Functions

Functions are blocks of code designed to perform specific tasks. They help organize code, promote reusability, and make programs more maintainable.

Function Declaration and Definition

A function has a declaration (prototype) and a definition:

// Function declaration (prototype)
returnType functionName(parameterType1 parameter1, parameterType2 parameter2);

// Function definition
returnType functionName(parameterType1 parameter1, parameterType2 parameter2) {
    // Function body
    // Code to perform the task
    return value; // Return a value of returnType
}

Example of a simple function:

// Function to add two integers
int add(int a, int b) {
    return a + b;
}

// Function usage
int result = add(5, 3); // result = 8

Function Parameters

Functions can take parameters, which are values passed to the function:

Pass by Value

void modifyValue(int x) {
    x = x * 2; // Modifies the local copy, not the original
}

int main() {
    int num = 10;
    modifyValue(num); // num is still 10 after the function call
    return 0;
}

Pass by Reference

void modifyValue(int &x) {
    x = x * 2; // Modifies the original value
}

int main() {
    int num = 10;
    modifyValue(num); // num becomes 20 after the function call
    return 0;
}

Return Values

Functions can return values using the return statement:

int getSquare(int x) {
    return x * x;
}

double calculateAverage(int a, int b, int c) {
    return (a + b + c) / 3.0;
}

Void Functions

Functions that don't return a value use the void return type:

void printMessage(string message) {
    cout << message << endl;
    // No return statement needed
}

Default Parameters

Functions can have default parameter values:

void setVolume(int level = 50) {
    cout << "Volume set to: " << level << endl;
}

int main() {
    setVolume();     // Uses default value: "Volume set to: 50"
    setVolume(75);   // Uses provided value: "Volume set to: 75"
    return 0;
}

Function Overloading

C++ allows multiple functions with the same name but different parameters:

int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

int main() {
    int sum1 = add(5, 3);         // Calls the int version
    double sum2 = add(3.5, 2.7);  // Calls the double version
    return 0;
}

Inline Functions

Inline functions can improve performance for small, frequently used functions:

inline int getMax(int a, int b) {
    return (a > b) ? a : b;
}

Recursive Functions

Functions that call themselves:

int factorial(int n) {
    if (n <= 1) {
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

int main() {
    int result = factorial(5); // 5! = 5 * 4 * 3 * 2 * 1 = 120
    return 0;
}

Function Pointers

Variables that store the address of functions:

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int main() {
    // Declare a function pointer
    int (*operation)(int, int);
    
    // Assign function address to pointer
    operation = add;
    cout << operation(5, 3) << endl; // Output: 8
    
    // Change function
    operation = subtract;
    cout << operation(5, 3) << endl; // Output: 2
    
    return 0;
}

Key points to remember:

  • Functions should have a clear, single purpose
  • Use meaningful function and parameter names
  • Keep functions relatively short and focused
  • Consider using pass by reference for large objects to improve performance
  • Function declarations (prototypes) should be placed in header files
  • Function definitions should be placed in source files
