Primitive Data Types - Basic Cheezein

JavaScript mein 7 primitive data types hain, inko samajh lo toh aadhi game jeet gaye:

Type Kya Hai? Example
String Text wala data 'Hello', "World", `Template literal`
Number Integer ya decimal numbers 42, 3.14, -273
BigInt Bahut bade numbers ke liye 9007199254740991n
Boolean Haan ya Na (true/false) true, false
Undefined Variable declare kiya par value nahi di undefined
Null Jaan bujhkar khaali chhoda hai null
Symbol Unique ID jaise value Symbol('description')

Variables in JavaScript - Containers Bro!

JavaScript mein variables declare karne ke 3 tarike hain, aur inme farak samajhna bahut zaroori hai:

// var use karna (purana tarika, avoid karo)
var name = "John";

// let use karna (naya cool tarika, variables ke liye jo change hote hain)
let age = 25;

// const use karna (constants ke liye jo kabhi change nahi hote)
const PI = 3.14159;
Yaad rakhna bro: Modern JavaScript mein var ki jagah let aur const use karo. var toh purani technology hai, dinosaur ke zamane ki! ü¶ñ

String Type - Text Wala Maal

Strings 3 tarike se bana sakte ho - single quotes, double quotes, ya backticks se:

// Single quotes
let firstName = 'John';

// Double quotes
let lastName = "Doe";

// Template literals (backticks) - isme expressions daal sakte ho
let greeting = `Hello, ${firstName} ${lastName}!`;
console.log(greeting);  // Output: Hello, John Doe!

Number Type - Digits Wala Scene

JavaScript mein ek hi number type hai, jo integers aur floating-point dono handle karta hai:

let integer = 42;
let float = 3.14;
let negative = -273;
let exponent = 2.998e8;  // Scientific notation: 2.998 √ó 10^8

// Special numeric values
let infinity = Infinity;  // Anant
let negInfinity = -Infinity;  // Negative anant
let notANumber = NaN;  // Not a Number - jab calculation fail ho jaye

Boolean Type - Haan ya Na

Boolean values sirf do ho sakti hain: true ya false. Simple!

let isActive = true;  // Haan
let isLoggedIn = false;  // Nahi

// Boolean expressions
let isGreater = 5 > 3;  // true, kyunki 5 bada hai 3 se
let isEqual = 4 === 4;  // true, dono equal hain
let isNotEqual = 3 !== 3;  // false, dono equal hain

Undefined aur Null - Khaali Containers

Ye dono types khaali values ko represent karte hain, par alag tarike se:

// Undefined: variable declare kiya par value assign nahi ki
let undefinedVar;
console.log(undefinedVar);  // Output: undefined

// Null: jaan bujhkar khaali value di hai
let emptyValue = null;
console.log(emptyValue);  // Output: null

Type Conversion - Ek Type se Doosre Type Mein Convert Karna

JavaScript automatic type conversion karta hai operations ke time, lekin kabhi kabhi unexpected results de sakta hai:

// String conversion
let value = true;
console.log(typeof value);  // "boolean"

value = String(value);      // ab value ek string "true" hai
console.log(typeof value);  // "string"

// Numeric conversion
console.log("6" / "2");     // 3, strings ko numbers mein convert kar diya
let str = "123";
let num = Number(str);      // explicit conversion
console.log(typeof num);    // "number"

// Boolean conversion
console.log(Boolean(1));    // true
console.log(Boolean(0));    // false
console.log(Boolean(""));   // false
console.log(Boolean("hello")); // true

Types Check Karna

Variable ka type check karne ke liye typeof operator use karo:

console.log(typeof "Hello");     // "string"
console.log(typeof 42);         // "number"
console.log(typeof true);       // "boolean"
console.log(typeof undefined);  // "undefined"
console.log(typeof null);       // "object" (ye JavaScript ka ek bug hai, lekin ab fix nahi ho sakta)
console.log(typeof {});         // "object"
console.log(typeof []);         // "object" (arrays bhi objects hi hain JavaScript mein)
console.log(typeof function(){}); // "function"
Important baat bro: typeof null return karta hai "object", jo ki technically galat hai, lekin ab ye JavaScript ka permanent bug ban gaya hai! üêõ

Yaad Rakhne Wali Baatein

  • JavaScript dynamically typed hai - variables ka type change ho sakta hai runtime pe
  • let use karo un variables ke liye jo change honge
  • const use karo un variables ke liye jo fixed rahenge
  • var se door raho modern JavaScript mein
  • Automatic type conversion se savdhaan rahein, unexpected results aa sakte hain
  • Hamesha strict equality operators (=== and !==) use karo taaki type conversion na ho
