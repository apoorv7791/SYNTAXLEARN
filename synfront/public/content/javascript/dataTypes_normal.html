JavaScript Data Types and Variables

JavaScript is a dynamically typed language, which means you don't need to declare the type of a variable when you create it. The type is determined automatically when the program runs.

Primitive Data Types

JavaScript has seven primitive data types:

Type Description Example
String Textual data 'Hello', "World", `Template literal`
Number Integer or floating-point numbers 42, 3.14, -273
BigInt Integers with arbitrary precision 9007199254740991n
Boolean Logical entity (true/false) true, false
Undefined Variable declared but not assigned undefined
Null Intentional absence of any value null
Symbol Unique and immutable value Symbol('description')

Variables in JavaScript

There are three ways to declare variables in JavaScript:

// Using var (function-scoped, older way)
var name = "John";

// Using let (block-scoped, recommended for variables that change)
let age = 25;

// Using const (block-scoped, for constants that don't change)
const PI = 3.14159;
Note: Modern JavaScript code typically uses let and const instead of var.

String Type

Strings in JavaScript can be created using single quotes, double quotes, or backticks (template literals):

// Single quotes
let firstName = 'John';

// Double quotes
let lastName = "Doe";

// Template literals (backticks) - allow embedded expressions
let greeting = `Hello, ${firstName} ${lastName}!`;
console.log(greeting);  // Output: Hello, John Doe!

Number Type

JavaScript has only one number type, which represents both integers and floating-point numbers:

let integer = 42;
let float = 3.14;
let negative = -273;
let exponent = 2.998e8;  // Scientific notation: 2.998 Ã— 10^8

// Special numeric values
let infinity = Infinity;
let negInfinity = -Infinity;
let notANumber = NaN;  // Result of undefined mathematical operations

Boolean Type

Boolean values represent logical entities and can have only two values: true or false.

let isActive = true;
let isLoggedIn = false;

// Boolean expressions
let isGreater = 5 > 3;  // true
let isEqual = 4 === 4;  // true
let isNotEqual = 3 !== 3;  // false

Undefined and Null

These two types represent the absence of a value, but in different ways:

// Undefined: variable declared but not assigned a value
let undefinedVar;
console.log(undefinedVar);  // Output: undefined

// Null: intentional absence of any object value
let emptyValue = null;
console.log(emptyValue);  // Output: null

Type Conversion

JavaScript often performs automatic type conversion (coercion) when operations involve different types:

// String conversion
let value = true;
console.log(typeof value);  // "boolean"

value = String(value);      // now value is a string "true"
console.log(typeof value);  // "string"

// Numeric conversion
console.log("6" / "2");     // 3, strings are converted to numbers
let str = "123";
let num = Number(str);      // explicit conversion
console.log(typeof num);    // "number"

// Boolean conversion
console.log(Boolean(1));    // true
console.log(Boolean(0));    // false
console.log(Boolean(""));   // false
console.log(Boolean("hello")); // true

Checking Types

You can check the type of a variable using the typeof operator:

console.log(typeof "Hello");     // "string"
console.log(typeof 42);         // "number"
console.log(typeof true);       // "boolean"
console.log(typeof undefined);  // "undefined"
console.log(typeof null);       // "object" (this is a known JavaScript quirk)
console.log(typeof {});         // "object"
console.log(typeof []);         // "object" (arrays are objects in JavaScript)
console.log(typeof function(){}); // "function"
Important: typeof null returns "object", which is considered a bug in JavaScript, but it's maintained for compatibility reasons.

Key Points to Remember

  • JavaScript is dynamically typed - variables can change types
  • Use let for variables that will change
  • Use const for variables that won't change
  • Avoid using var in modern JavaScript code
  • Be careful with automatic type conversion in operations
  • Always use strict equality operators (=== and !==) to avoid unexpected type conversions
