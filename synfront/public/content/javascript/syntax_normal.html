Basic Syntax and Structure

JavaScript is a flexible language with syntax influenced by C and Java. Understanding the basic syntax and structure is essential for writing correct and efficient JavaScript code.

JavaScript Statements

JavaScript code consists of statements that are executed by the browser or Node.js runtime. Statements are separated by semicolons (;):

// This is a single statement
console.log("Hello, World!");

// Multiple statements
let x = 5; let y = 10; let z = x + y;

While semicolons are technically optional in many cases due to automatic semicolon insertion (ASI), it's considered good practice to include them to avoid potential issues.

Comments

Comments are used to add notes to your code and are ignored by the JavaScript engine:

// This is a single-line comment

/* This is a
   multi-line comment */

Case Sensitivity

JavaScript is case-sensitive. Variables, function names, and other identifiers must be typed with consistent capitalization:

let name = "John";
let Name = "Jane";  // Different variable than name

console.log(name);  // Outputs: John
console.log(Name);  // Outputs: Jane

Whitespace and Line Breaks

JavaScript ignores whitespace (spaces, tabs, newlines) between statements. This allows you to format your code for readability:

// These are equivalent
let sum = x + y;

let sum=x+y;

Code Blocks

Code blocks are defined by curly braces {} and are used to group statements together:

// If statement with a code block
if (condition) {
    statement1;
    statement2;
    statement3;
}

Identifiers

Identifiers are names used for variables, functions, classes, etc. They must follow these rules:

  • Can contain letters, digits, underscores (_), and dollar signs ($)
  • Must begin with a letter, underscore (_), or dollar sign ($)
  • Cannot use reserved words (like JavaScript keywords)
  • Are case-sensitive
// Valid identifiers
let firstName = "John";
let _privateVar = 42;
let $price = 19.99;
let counter1 = 0;

// Invalid identifiers
// let 1stPlace = "Gold";  // Cannot start with a digit
// let for = "loop";       // Cannot use reserved word

Variables

Variables are used to store data values. In modern JavaScript, there are three ways to declare variables:

// Using var (older way, function-scoped)
var age = 30;

// Using let (block-scoped, can be reassigned)
let score = 75;

// Using const (block-scoped, cannot be reassigned)
const PI = 3.14159;

It's recommended to use const by default, and only use let when you need to reassign a variable. Avoid var in modern JavaScript.

Data Types

JavaScript has several built-in data types:

Primitive Types

// Number
let age = 30;
let price = 19.99;

// String
let name = "John";
let greeting = 'Hello';

// Boolean
let isActive = true;
let isCompleted = false;

// Undefined
let undefinedVar;

// Null
let emptyValue = null;

// Symbol (ES6)
let uniqueId = Symbol("id");

// BigInt (ES2020)
let bigNumber = 9007199254740991n;

Reference Types

// Object
let person = {
    firstName: "John",
    lastName: "Doe",
    age: 30
};

// Array (special type of object)
let colors = ["red", "green", "blue"];

// Function (also a special type of object)
function greet() {
    console.log("Hello!");
}

Operators

JavaScript supports various operators for performing operations on values:

Arithmetic Operators

let a = 10;
let b = 5;

let sum = a + b;        // Addition: 15
let difference = a - b; // Subtraction: 5
let product = a * b;    // Multiplication: 50
let quotient = a / b;   // Division: 2
let remainder = a % b;  // Modulus (remainder): 0
let power = a ** b;     // Exponentiation: 100000

Assignment Operators

let x = 10;  // Assign 10 to x

x += 5;      // Same as: x = x + 5
x -= 3;      // Same as: x = x - 3
x *= 2;      // Same as: x = x * 2
x /= 4;      // Same as: x = x / 4
x %= 3;      // Same as: x = x % 3

Comparison Operators

let a = 5;
let b = 10;
let c = "5";

a == c;   // Equal to (value only): true
a === c;  // Strict equal to (value and type): false
a != b;   // Not equal to: true
a !== c;  // Strict not equal to: true
a < b;    // Less than: true
a > b;    // Greater than: false
a <= b;   // Less than or equal to: true
a >= b;   // Greater than or equal to: false

Logical Operators

let isAdult = true;
let hasPermission = false;

isAdult && hasPermission;  // Logical AND: false
isAdult || hasPermission;  // Logical OR: true
!isAdult;                  // Logical NOT: false

Expressions

An expression is any valid unit of code that resolves to a value:

// Arithmetic expression
let result = 5 * (10 - 3);

// String expression
let fullName = firstName + " " + lastName;

// Logical expression
let canAccess = isAdult && hasPermission;

// Assignment expression
let x = y = z = 5;  // Assigns 5 to z, then to y, then to x

Conditional Statements

Conditional statements are used to perform different actions based on different conditions:

if...else Statement

let age = 18;

if (age >= 18) {
    console.log("You are an adult.");
} else {
    console.log("You are a minor.");
}

switch Statement

let day = 2;
let dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    default:
        dayName = "Unknown";
}

console.log(dayName);  // Outputs: Tuesday

Loops

Loops are used to execute a block of code multiple times:

for Loop

// Print numbers from 1 to 5
for (let i = 1; i <= 5; i++) {
    console.log(i);
}

while Loop

let i = 1;

// Print numbers from 1 to 5
while (i <= 5) {
    console.log(i);
    i++;
}

do...while Loop

let i = 1;

// Print numbers from 1 to 5
do {
    console.log(i);
    i++;
} while (i <= 5);

for...of Loop (for iterables like arrays)

const colors = ["red", "green", "blue"];

// Print each color
for (const color of colors) {
    console.log(color);
}

for...in Loop (for object properties)

const person = {
    name: "John",
    age: 30,
    job: "Developer"
};

// Print each property
for (const key in person) {
    console.log(key + ": " + person[key]);
}

Functions

Functions are blocks of code designed to perform a particular task and can be reused throughout your code:

// Function declaration
function greet(name) {
    return "Hello, " + name + "!";
}

// Function expression
const add = function(a, b) {
    return a + b;
};

// Arrow function (ES6)
const multiply = (a, b) => a * b;

// Calling functions
console.log(greet("John"));     // Outputs: Hello, John!
console.log(add(5, 3));         // Outputs: 8
console.log(multiply(4, 2));    // Outputs: 8

Error Handling

JavaScript provides a way to handle errors using try...catch blocks:

try {
    // Code that might throw an error
    let result = someUndefinedVariable * 2;
} catch (error) {
    // Code to handle the error
    console.error("An error occurred:", error.message);
} finally {
    // Code that will run regardless of whether an error occurred
    console.log("This will always execute");
}

Key points to remember:

  • JavaScript is case-sensitive
  • Statements are separated by semicolons (;)
  • Comments are written with // for single-line and /* */ for multi-line
  • Code blocks are enclosed in curly braces {}
  • Use const by default, let when you need to reassign, and avoid var
  • JavaScript has both primitive and reference data types
  • Always use strict equality (===) and inequality (!==) operators to avoid type coercion issues
  • Functions are first-class objects in JavaScript and can be assigned to variables, passed as arguments, etc.
