Objects in JavaScript - JavaScript Ke Objects

Objects key-value pairs ke collections hain, jahan values properties (data) ya methods (functions) ho sakte hain - bilkul tumhare phone ki tarah, jisme features (properties) aur functions (methods) hote hain!

Creating Objects - Objects Banane Ke Tarike

JavaScript mein objects banane ke kai tarike hain:

Object Literals - Seedha Object Banana
// Object literal syntax - sabse easy tarika
const person = {
    firstName: "Rahul",
    lastName: "Sharma",
    age: 25,
    greet: function() {
        return `Hello bro, myself ${this.firstName} ${this.lastName}`;
    }
};

console.log(person.firstName);  // Output: "Rahul"
console.log(person.greet());    // Output: "Hello bro, myself Rahul Sharma"
Constructor Functions - Factory Ki Tarah
// Constructor function - ek template ki tarah
function Person(firstName, lastName, age) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.age = age;
    this.greet = function() {
        return `Hello bro, myself ${this.firstName} ${this.lastName}`;
    };
}

const rahul = new Person("Rahul", "Sharma", 25);
console.log(rahul.firstName);  // Output: "Rahul"
console.log(rahul.greet());    // Output: "Hello bro, myself Rahul Sharma"

Prototypes and Inheritance - Virasat Ki Kahani

JavaScript mein inheritance prototype-based hota hai. Har object ka ek prototype hota hai, jisse wo properties aur methods inherit karta hai - bilkul family inheritance ki tarah!

Prototype Chain - Prototype Ki Chain

// Constructor function
function Person(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
}

// Prototype mein method add karna - memory efficient hai ye!
Person.prototype.getFullName = function() {
    return `${this.firstName} ${this.lastName}`;
};

const rahul = new Person("Rahul", "Sharma");
console.log(rahul.getFullName());  // Output: "Rahul Sharma"
Pro tip bro: Methods ko prototype mein add karna best practice hai, kyunki isse memory save hoti hai! Har instance ke paas apna alag method nahi hota, sab same prototype se share karte hain - ekdam desi jugaad! ðŸ’¯

Inheritance with Prototypes - Purane Tarike Se Inheritance

// Parent constructor
function Person(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
}

Person.prototype.greet = function() {
    return `Hello bro, myself ${this.firstName} ${this.lastName}`;
};

// Child constructor
function Employee(firstName, lastName, jobTitle) {
    // Parent constructor ko call karo
    Person.call(this, firstName, lastName);
    this.jobTitle = jobTitle;
}

// Person ke prototype se inherit karo
Employee.prototype = Object.create(Person.prototype);
Employee.prototype.constructor = Employee;

// Employee ke liye special method
Employee.prototype.introduce = function() {
    return `${this.greet()}. I work as a ${this.jobTitle}.`;
};

const rahul = new Employee("Rahul", "Sharma", "Software Developer");
console.log(rahul.introduce());  // Output: "Hello bro, myself Rahul Sharma. I work as a Software Developer."

Inheritance with ES6 Classes - Modern Tarike Se Inheritance

// Parent class
class Person {
    constructor(firstName, lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }

    greet() {
        return `Hello bro, myself ${this.firstName} ${this.lastName}`;
    }
}

// Child class extending Person
class Employee extends Person {
    constructor(firstName, lastName, jobTitle) {
        super(firstName, lastName);  // Parent constructor ko call karo
        this.jobTitle = jobTitle;
    }

    introduce() {
        return `${this.greet()}. I work as a ${this.jobTitle}.`;
    }
}

const rahul = new Employee("Rahul", "Sharma", "Software Developer");
console.log(rahul.introduce());  // Output: "Hello bro, myself Rahul Sharma. I work as a Software Developer."

Encapsulation - Private Data Rakhna

Encapsulation matlab data aur methods ko bundle karna, aur kuch components ko private rakhna - bilkul apne personal diary ki tarah, jo sirf aap hi padh sakte ho!

Private Properties and Methods with Closures - Closures Se Privacy

// Closures ka use karke private properties banao
function Person(firstName, lastName) {
    // Private properties - bahar se access nahi kar sakte
    const _firstName = firstName;
    const _lastName = lastName;

    // Public methods jo private properties ko access kar sakte hain
    this.getFullName = function() {
        return `${_firstName} ${_lastName}`;
    };

    this.greet = function() {
        return `Hello bro, myself ${this.getFullName()}`;
    };
}

const rahul = new Person("Rahul", "Sharma");
console.log(rahul.getFullName());  // Output: "Rahul Sharma"
// console.log(rahul._firstName);  // Output: undefined (private property hai bro!)

Private Fields (ES2022) - Modern Tarike Se Privacy

// Private fields ka use (modern JavaScript)
class Person {
    // Private fields (# se start hote hain)
    #firstName;

        this.#firstName = firstName;

        return `${this.#firstName} ${this.#lastName}`;
    }

        return `Hello bro, myself ${this.getFullName()}`;
    }
}

const rahul = new Person("Rahul", "Sharma");

// console.log(rahul.#firstName);  // Error: Private field hai bro, bahar se access nahi kar sakte!

Polymorphism - Ek Interface, Multiple Forms

Polymorphism matlab ek hi method ko alag-alag classes mein alag-alag tarike se implement karna - jaise ki alag-alag log ek hi sawal ka alag-alag jawab dete hain!

// Parent class
class Animal {
    speak() {
        return "Animal makes a sound";
    }
}

// Child classes overriding the speak method
class Dog extends Animal {
    speak() {
        return "Dog bolta hai: Woof Woof!";
    }
}

class Cat extends Animal {
    speak() {
        return "Cat bolti hai: Meow Meow!";
    }
}

// Function that works with any Animal
function makeAnimalSpeak(animal) {
    console.log(animal.speak());
}

const animal = new Animal();
const dog = new Dog();
const cat = new Cat();

makeAnimalSpeak(animal);  // Output: "Animal makes a sound"
makeAnimalSpeak(dog);     // Output: "Dog bolta hai: Woof Woof!"
makeAnimalSpeak(cat);     // Output: "Cat bolti hai: Meow Meow!"

Getters and Setters - Properties Ki Tarah Methods

Getters aur setters se aap methods ko properties ki tarah access kar sakte hain - matlab method ko call karne ke liye parentheses () ki zaroorat nahi!

// Using getters and setters with classes
class Person {
    constructor(firstName, lastName) {
        this._firstName = firstName;
        this._lastName = lastName;
    }

    // Getter for fullName
    get fullName() {
        return `${this._firstName} ${this._lastName}`;
    }

    // Setter for fullName
    set fullName(name) {
        const parts = name.split(' ');
        this._firstName = parts[0];
        this._lastName = parts[1] || '';
    }
}

const rahul = new Person("Rahul", "Sharma");
console.log(rahul.fullName);  // Output: "Rahul Sharma" (method ko property ki tarah call kiya!)

rahul.fullName = "Rohit Verma";  // Value assign karne se setter call hota hai
console.log(rahul.fullName);  // Output: "Rohit Verma"

Static Methods and Properties - Class Level Methods

Static methods aur properties class se belong karte hain, instances se nahi - matlab inko use karne ke liye object banane ki zaroorat nahi!

// Using static methods and properties
class MathUtils {
    static PI = 3.14159;

    static square(x) {
        return x * x;
    }

    static cube(x) {
        return x * x * x;
    }
}

console.log(MathUtils.PI);        // Output: 3.14159
console.log(MathUtils.square(3));  // Output: 9
console.log(MathUtils.cube(3));    // Output: 27

  • JavaScript prototype-based hai, par ES6 classes ne syntax ko easy bana diya hai
  • Objects banane ke 4 tarike hain: object literals, constructor functions, classes, ya Object.create()
  • JavaScript mein inheritance prototype chain ke through kaam karta hai
  • ES6 classes ne inheritance ko clean bana diya hai extends keyword aur super() method ke saath
  • Private data rakhne ke liye closures ya private fields (#) use kar sakte hain
  • Polymorphism method overriding ke through implement hota hai
  • Getters aur setters se methods ko properties ki tarah use kar sakte hain
  • Static methods aur properties class level par hote hain, instance level par nahi
