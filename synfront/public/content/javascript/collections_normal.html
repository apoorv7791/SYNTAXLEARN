Arrays and Objects in JavaScript

JavaScript provides two primary data structures for storing collections of data: Arrays and Objects. Understanding these structures is crucial for effective JavaScript programming.

Arrays

Arrays are ordered collections of values. They can store multiple values of any type in a single variable.

Creating Arrays

// Array literal syntax (preferred)
const fruits = ["apple", "banana", "orange"];

// Array constructor
const numbers = new Array(1, 2, 3, 4, 5);

// Empty array
const emptyArray = [];

// Array with mixed data types
const mixedArray = [1, "hello", true, null, { name: "John" }, [1, 2, 3]];

Accessing Array Elements

const fruits = ["apple", "banana", "orange"];

// Accessing by index (zero-based)
console.log(fruits[0]);  // Output: "apple"
console.log(fruits[1]);  // Output: "banana"
console.log(fruits[2]);  // Output: "orange"

// Length property
console.log(fruits.length);  // Output: 3

// Accessing last element
console.log(fruits[fruits.length - 1]);  // Output: "orange"

Modifying Arrays

const fruits = ["apple", "banana", "orange"];

// Changing an element
fruits[1] = "grape";
console.log(fruits);  // Output: ["apple", "grape", "orange"]

// Adding elements to the end
fruits.push("kiwi");
console.log(fruits);  // Output: ["apple", "grape", "orange", "kiwi"]

// Removing the last element
const lastFruit = fruits.pop();
console.log(lastFruit);  // Output: "kiwi"
console.log(fruits);     // Output: ["apple", "grape", "orange"]

// Adding elements to the beginning
fruits.unshift("strawberry");
console.log(fruits);  // Output: ["strawberry", "apple", "grape", "orange"]

// Removing the first element
const firstFruit = fruits.shift();
console.log(firstFruit);  // Output: "strawberry"
console.log(fruits);      // Output: ["apple", "grape", "orange"]

Common Array Methods

Method Description Example
concat() Joins two or more arrays and returns a new array const newArray = array1.concat(array2, array3);
slice() Returns a shallow copy of a portion of an array const subArray = array.slice(1, 3);
splice() Changes an array by removing or replacing elements and/or adding new elements array.splice(2, 1, 'new');
indexOf() Returns the first index at which a given element can be found const index = array.indexOf('apple');
includes() Determines whether an array includes a certain value const hasApple = array.includes('apple');
join() Joins all elements of an array into a string const str = array.join(', ');
reverse() Reverses the order of the elements in an array array.reverse();
sort() Sorts the elements of an array array.sort();

Iterating Over Arrays

const fruits = ["apple", "banana", "orange"];

// for loop
for (let i = 0; i < fruits.length; i++) {
    console.log(fruits[i]);
}

// for...of loop (recommended for arrays)
for (const fruit of fruits) {
    console.log(fruit);
}

// forEach method
fruits.forEach(function(fruit, index) {
    console.log(`${index}: ${fruit}`);
});

Array Methods for Transformation

const numbers = [1, 2, 3, 4, 5];

// map: creates a new array with the results of calling a function on every element
const doubled = numbers.map(function(num) {
    return num * 2;
});
console.log(doubled);  // Output: [2, 4, 6, 8, 10]

// filter: creates a new array with elements that pass the test
const evenNumbers = numbers.filter(function(num) {
    return num % 2 === 0;
});
console.log(evenNumbers);  // Output: [2, 4]

// reduce: applies a function against an accumulator and each element to reduce to a single value
const sum = numbers.reduce(function(total, num) {
    return total + num;
}, 0);
console.log(sum);  // Output: 15

// find: returns the first element that satisfies the provided testing function
const firstEven = numbers.find(function(num) {
    return num % 2 === 0;
});
console.log(firstEven);  // Output: 2

// some: tests whether at least one element passes the test
const hasEven = numbers.some(function(num) {
    return num % 2 === 0;
});
console.log(hasEven);  // Output: true

// every: tests whether all elements pass the test
const allEven = numbers.every(function(num) {
    return num % 2 === 0;
});
console.log(allEven);  // Output: false

Objects

Objects are collections of key-value pairs, where keys are strings (or Symbols) and values can be any data type.

Creating Objects

// Object literal syntax (preferred)
const person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    isEmployed: true
};

// Object constructor
const car = new Object();
car.make = "Toyota";
car.model = "Camry";
car.year = 2020;

// Empty object
const emptyObject = {};

Accessing Object Properties

const person = {
    firstName: "John",
    lastName: "Doe",
    age: 30
};

// Dot notation
console.log(person.firstName);  // Output: "John"

// Bracket notation (useful when property name is dynamic or has special characters)
console.log(person["lastName"]);  // Output: "Doe"

const propertyName = "age";
console.log(person[propertyName]);  // Output: 30

Modifying Objects

const person = {
    firstName: "John",
    lastName: "Doe"
};

// Adding properties
person.age = 30;
person["isEmployed"] = true;

// Updating properties
person.firstName = "Jane";

// Deleting properties
delete person.isEmployed;

console.log(person);  // Output: { firstName: "Jane", lastName: "Doe", age: 30 }

Object Methods

const person = {
    firstName: "John",
    lastName: "Doe",
    // Method definition
    getFullName: function() {
        return `${this.firstName} ${this.lastName}`;
    },
    // Shorthand method definition (ES6)
    greet() {
        return `Hello, my name is ${this.getFullName()}`;
    }
};

console.log(person.getFullName());  // Output: "John Doe"
console.log(person.greet());        // Output: "Hello, my name is John Doe"

Working with Object Properties

const person = {
    firstName: "John",
    lastName: "Doe",
    age: 30
};

// Checking if a property exists
console.log("firstName" in person);  // Output: true
console.log(person.hasOwnProperty("gender"));  // Output: false

// Getting all keys
const keys = Object.keys(person);
console.log(keys);  // Output: ["firstName", "lastName", "age"]

// Getting all values
const values = Object.values(person);
console.log(values);  // Output: ["John", "Doe", 30]

// Getting all entries (key-value pairs)
const entries = Object.entries(person);
console.log(entries);  // Output: [["firstName", "John"], ["lastName", "Doe"], ["age", 30]]

Iterating Over Objects

const person = {
    firstName: "John",
    lastName: "Doe",
    age: 30
};

// for...in loop (for objects)
for (const key in person) {
    if (person.hasOwnProperty(key)) {  // Ensure it's not from the prototype chain
        console.log(`${key}: ${person[key]}`);
    }
}

// Using Object.keys()
Object.keys(person).forEach(function(key) {
    console.log(`${key}: ${person[key]}`);
});

// Using Object.entries()
Object.entries(person).forEach(function([key, value]) {
    console.log(`${key}: ${value}`);
});

Object Destructuring

const person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    address: {
        street: "123 Main St",
        city: "New York",
        country: "USA"
    }
};

// Basic destructuring
const { firstName, lastName } = person;
console.log(firstName);  // Output: "John"
console.log(lastName);   // Output: "Doe"

// Destructuring with new variable names
const { firstName: fName, lastName: lName } = person;
console.log(fName);  // Output: "John"
console.log(lName);  // Output: "Doe"

// Nested destructuring
const { address: { city, country } } = person;
console.log(city);     // Output: "New York"
console.log(country);  // Output: "USA"

// Default values
const { gender = "Unknown" } = person;
console.log(gender);  // Output: "Unknown"

Spread Operator with Objects

const person = {
    firstName: "John",
    lastName: "Doe"
};

// Creating a new object with all properties from person plus additional ones
const employee = {
    ...person,
    jobTitle: "Developer",
    company: "Acme Inc."
};
console.log(employee);
// Output: { firstName: "John", lastName: "Doe", jobTitle: "Developer", company: "Acme Inc." }

// Merging objects
const personalInfo = { age: 30, gender: "Male" };
const contactInfo = { email: "john@example.com", phone: "123-456-7890" };

const completeInfo = { ...person, ...personalInfo, ...contactInfo };
console.log(completeInfo);
// Output: { firstName: "John", lastName: "Doe", age: 30, gender: "Male", email: "john@example.com", phone: "123-456-7890" }

Maps and Sets

JavaScript also provides more specialized collection types: Map and Set.

Map

Map is a collection of key-value pairs where keys can be of any type.

// Creating a Map
const userRoles = new Map();

// Adding entries
userRoles.set("John", "Admin");
userRoles.set("Jane", "Editor");
userRoles.set("Bob", "Subscriber");

// Getting values
console.log(userRoles.get("John"));  // Output: "Admin"

// Checking if a key exists
console.log(userRoles.has("Jane"));  // Output: true

// Size of the Map
console.log(userRoles.size);  // Output: 3

// Deleting an entry
userRoles.delete("Bob");

// Iterating over a Map
for (const [user, role] of userRoles) {
    console.log(`${user}: ${role}`);
}

Set

Set is a collection of unique values of any type.

// Creating a Set
const uniqueNumbers = new Set([1, 2, 3, 4, 4, 5, 5]);  // Duplicates are automatically removed

// Adding values
uniqueNumbers.add(6);
uniqueNumbers.add(1);  // Ignored because 1 already exists

// Checking if a value exists
console.log(uniqueNumbers.has(3));  // Output: true

// Size of the Set
console.log(uniqueNumbers.size);  // Output: 6

// Deleting a value
uniqueNumbers.delete(4);

// Iterating over a Set
for (const num of uniqueNumbers) {
    console.log(num);
}

Key Points to Remember

  • Arrays are ordered collections accessed by index
  • Objects are unordered collections of key-value pairs
  • Use arrays when the order of elements matters or when you need to store a list of similar items
  • Use objects when you need to associate values with specific keys or create complex data structures
  • Array methods like map(), filter(), and reduce() are powerful tools for data transformation
  • Object destructuring and the spread operator provide convenient ways to work with object data
  • Maps are similar to objects but allow keys of any type and maintain insertion order
  • Sets are useful when you need to store unique values and check for existence quickly
