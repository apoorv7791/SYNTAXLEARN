Bro, class banane ke liye class keyword use karte hai:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

__init__ Method Kya Hai?

Bro, __init__ constructor hai - jab bhi new object banate hai, ye automatically call hota hai. Isme hum object ki initial properties set karte hai.

self Parameter Kya Hai?

Bro, self matlab "khud" - ye object ko refer karta hai. Isse hum object ke variables aur methods access kar sakte hai.

Class Variables vs. Instance Variables üîÑ

class Student:
    # Class variable - sabke liye same
    school = "Coding Wala School"
    
    def __init__(self, name, grade):
        # Instance variables - har student ke liye alag
        self.name = name
        self.grade = grade

topper = Student("Riya", "A+")
backbencher = Student("Vicky", "C")

print(topper.school)      # Class variable
print(backbencher.school)
print(topper.name)        # Instance variable
print(backbencher.name)

Output:
Coding Wala School
Coding Wala School
Riya
Vicky

Inheritance - Baap Ka Maal Beta Ka üòÇ

Bro, inheritance matlab parent class se properties aur methods inherit karna:

class Animal:
    def __init__(self, species):
        self.species = species
    
    def sound(self):
        print("Koi awaaz")

class Dog(Animal):
    def __init__(self, name):
        # Parent class ko call kiya
        super().__init__("Kutta")
        self.name = name
    
    # Parent method ko override kiya
    def sound(self):
        print("Bhow Bhow!")

tommy = Dog("Tommy")
print(f"{tommy.name} ek {tommy.species} hai")
tommy.sound()

Output:
Tommy ek Kutta hai
Bhow Bhow!

Multiple Inheritance - Double Dhamaka! üí•

Bro, Python mein ek child class multiple parents se inherit kar sakta hai:

class Flyable:
    def fly(self):
        print("Udaan bharta hun!")

class Swimmable:
    def swim(self):
        print("Tairta hun!")

class Duck(Flyable, Swimmable):
    def quack(self):
        print("Quack Quack!")

donald = Duck()
donald.fly()    # Flyable se
donald.swim()   # Swimmable se
donald.quack()  # Khud ka

Tairta hun!
Quack Quack!

Encapsulation - Data Ko Safe Rakhna üîí

Bro, encapsulation matlab data ko private rakhna. Python mein naming conventions se hota hai:

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner        # Public - koi bhi access kar sakta hai
        self._balance = balance   # Protected - sirf warning hai
        self.__id = 12345         # Private - bahar se access nahi kar sakte
    
    def deposit(self, amount):
        self._balance += amount
        return self._balance
    
    def get_balance(self):
        return self._balance

account = BankAccount("Raj")
account.deposit(5000)
print(f"Balance: ‚Çπ{account.get_balance()}")
# print(account.__id)  # Error aayega!

Output:
Balance: ‚Çπ5000

Polymorphism - Ek Cheez, Alag Alag Behavior ü¶é

Bro, polymorphism matlab different classes ke objects ko same tarike se use karna:

class Shape:
    def area(self):
        pass

class Rectangle(Shape):
    def __init__(self, length, width):
        self.length = length
        self.width = width
    
    def area(self):
        return self.length * self.width

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return 3.14 * self.radius * self.radius

shapes = [Rectangle(5, 4), Circle(7)]
for shape in shapes:
    print(f"Area: {shape.area()}")

Output:
Area: 20
Area: 153.86

Special Methods - Dunder Methods üîÆ

Bro, Python mein special methods hote hai jo double underscore se start aur end hote hai:

class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages
    
    def __str__(self):
        return f"{self.title} by {self.author}"
    
    def __len__(self):
        return self.pages
    
    def __del__(self):
        print(f"Book {self.title} delete ho gaya!")

book = Book("Python Seekho", "Raj Kumar", 200)
print(book)          # __str__ call hoga
print(len(book))     # __len__ call hoga
del book             # __del__ call hoga

Output:
Python Seekho by Raj Kumar
200
Book Python Seekho delete ho gaya!

Pro Tip: Bro, inheritance se better hai composition use karna - matlab dusre objects ko as attributes rakhna. Isse code flexible rehta hai!

Property Decorators - Getters aur Setters üéÄ

Bro, Python mein property decorators se getters aur setters implement kar sakte hai:

class Temperature:
    def __init__(self, celsius=0):
        self._celsius = celsius
    
    @property
    def celsius(self):
        return self._celsius
    
    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("Bhai, absolute zero se neeche temperature nahi ho sakta!")
        self._celsius = value
    
    @property
    def fahrenheit(self):
        return self._celsius * 9/5 + 32
    
    @fahrenheit.setter
    def fahrenheit(self, value):
        self.celsius = (value - 32) * 5/9

temp = Temperature(25)
print(f"{temp.celsius}¬∞C = {temp.fahrenheit}¬∞F")
temp.fahrenheit = 68
print(f"{temp.celsius}¬∞C = {temp.fahrenheit}¬∞F")

Output:
25¬∞C = 77.0¬∞F
20.0¬∞C = 68.0¬∞F
