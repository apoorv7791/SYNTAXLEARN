Common Exceptions in Python üö®

Bro, Python mein bahut saare built-in exceptions hai. Yahan kuch common wale hai:

  • SyntaxError: Jab code ka syntax galat ho
  • NameError: Jab variable defined na ho
  • TypeError: Jab operation galat type pe kiya jaye
  • ValueError: Jab value inappropriate ho
  • IndexError: Jab index range se bahar ho
  • KeyError: Jab key dictionary mein na ho
  • FileNotFoundError: Jab file na mile
  • ZeroDivisionError: Jab zero se divide kiya jaye
  • ImportError: Jab module import na ho sake
  • AttributeError: Jab attribute object mein na ho

Basic Exception Handling üõ°Ô∏è

Bro, try-except block se exceptions handle karte hai:

try:
    # Ye code exception raise kar sakta hai
    result = 10 / 0
except ZeroDivisionError:
    # Exception handle karne ka code
    print("Error: Bhai zero se divide nahi kar sakte!")

# Program execution continue hota hai
print("Program chal raha hai!")

Output:
Error: Bhai zero se divide nahi kar sakte!
Program chal raha hai!

Multiple Exceptions Handle Karna üß©

# Ek saath multiple exceptions bhi catch kar sakte ho

    # Kuch code jo exceptions raise kar sakta hai

else Clause Kya Hai? üéØ

Bro, else clause tab execute hota hai jab try block mein koi exception nahi aati:

try:

    result = 100 / num

    print("Error: Bhai ye to number hi nahi hai!")

    print("Error: Zero se divide? Dimag theek hai?")
else:
    # Ye tab execute hota hai jab koi exception nahi aati

    print("Sab kuch theek hai, koi error nahi!")

finally Clause Kya Hai? üîÑ

Bro, finally clause hamesha execute hota hai, chahe exception aaye ya na aaye:

try:
    file = open("example.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("Error: Bhai file to hai hi nahi!")
finally:
    # Ye hamesha execute hota hai, chahe exception aaye ya na aaye
    # Cleanup operations ke liye perfect hai, jaise file close karna
    if 'file' in locals() and not file.closed:
        file.close()
        print("File band kar di!")

def validate_age(age):

Custom Exceptions Banana üõ†Ô∏è

Bro, apne khud ke exceptions bhi bana sakte ho built-in Exception class se inherit karke:

class PaiseKamException(Exception):
    """Jab withdrawal balance se zyada ho tab raise hota hai"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        self.deficit = amount - balance
        message = f"‚Çπ{amount} nahi nikal sakte. Balance hai ‚Çπ{balance}, ‚Çπ{self.deficit} kam hai!"
        super().__init__(message)

class BankAccount:
    def __init__(self, balance=0):
        self.balance = balance
        
    def withdraw(self, amount):
        if amount > self.balance:
            raise PaiseKamException(self.balance, amount)
        self.balance -= amount
        return self.balance

# Custom exception use karna
try:
    account = BankAccount(1000)
    account.withdraw(1500)
except PaiseKamException as e:
    print(f"Error: {e}")
    print(f"Bhai, ‚Çπ{e.deficit} aur chahiye transaction ke liye!")

Output:
Error: ‚Çπ1500 nahi nikal sakte. Balance hai ‚Çπ1000, ‚Çπ500 kam hai!
Bhai, ‚Çπ500 aur chahiye transaction ke liye!

Context Managers aur with Statement üß∞

Bro, with statement se resources clean tarike se handle kar sakte ho:

# Without context manager
try:
    file = open("example.txt", "r")
    content = file.read()
finally:
    file.close()

# With context manager
with open("example.txt", "r") as file:
    content = file.read()
# File automatically close ho jati hai jab block exit hota hai

Custom Context Managers Banana üèóÔ∏è

Bro, apne custom context managers bhi bana sakte ho __enter__ aur __exit__ methods use karke:

class DatabaseConnection:
    def __init__(self, connection_string):
        self.connection_string = connection_string
        self.connection = None
    
    def __enter__(self):
        print(f"Database se connect kar raha hun: {self.connection_string}")
        # Real scenario mein, ye database connection establish karega
        self.connection = "Connected"
        return self.connection
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        print("Database connection band kar raha hun")
        # Real scenario mein, ye connection close karega
        self.connection = None
        # Return False to propagate exceptions, True to suppress them
        return False

# Custom context manager use karna
try:
    with DatabaseConnection("mysql://localhost:3306/mydb") as conn:
        print(f"Connection status: {conn}")
        # Error simulate karo
        raise ValueError("Database query fail ho gaya")
except ValueError as e:
    print(f"Exception pakda: {e}")

Output:
Database se connect kar raha hun: mysql://localhost:3306/mydb
Connection status: Connected
Database connection band kar raha hun
Exception pakda: Database query fail ho gaya

contextlib Module Kya Hai? üß©

Bro, Python ka contextlib module context managers ke liye utilities provide karta hai:

from contextlib import contextmanager

@contextmanager
def file_manager(filename, mode):
    try:
        file = open(filename, mode)
        yield file
    finally:
        file.close()

# Context manager use karna
with file_manager("example.txt", "r") as file:
    content = file.read()
    print(content)

Exception Handling Best Practices üíØ

Kya Karna Chahiye:

  • Specific exceptions catch karo, generic nahi
  • Try block ko jitna ho sake chota rakho
  • Cleanup operations ke liye finally use karo
  • Document karo ki tumhare functions kaun se exceptions raise kar sakte hai
  • Domain-specific errors ke liye custom exceptions use karo
  • Meaningful error messages do

Kya Nahi Karna Chahiye:

  • Bina exception type specify kiye except: use nahi karna chahiye (ye sab exceptions catch karega, KeyboardInterrupt bhi)
  • Exceptions ko properly handle kiye bina ignore nahi karna chahiye
  • Normal situations mein flow control ke liye exceptions use nahi karna chahiye
  • Aise exceptions catch nahi karne chahiye jinhe handle karna nahi aata

Assertions Kya Hote Hai? ‚úÖ

Bro, assertions debugging ke liye hote hai - ye ek condition test karte hai aur agar condition false hai to AssertionError raise karte hai:

def calculate_average(numbers):
    assert len(numbers) > 0, "Empty list ka average nahi nikal sakte bhai!"
    return sum(numbers) / len(numbers)

try:
    avg = calculate_average([])
except AssertionError as e:
    print(f"Error: {e}")

Output:
Error: Empty list ka average nahi nikal sakte bhai!

Note: Bro, assertions ko -O flag se disable kiya ja sakta hai Python run karte time, isliye inhe production code mein input validation ke liye use nahi karna chahiye. Explicit checks use karo aur appropriate exceptions raise karo.
