SQL Data Types

SQL databases use various data types to store different kinds of information. Understanding these data types is crucial for designing efficient database schemas and writing effective queries.

Numeric Data Types

Data Type Description Storage Size Range
INTEGER or INT Whole numbers without decimals 4 bytes -2,147,483,648 to 2,147,483,647
SMALLINT Small range integer 2 bytes -32,768 to 32,767
BIGINT Large range integer 8 bytes -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
DECIMAL(p,s) or NUMERIC(p,s) Exact numeric with specified precision (p) and scale (s) Variable Depends on precision
FLOAT Approximate numeric values 4 or 8 bytes Varies by implementation
REAL Single precision floating-point 4 bytes Varies by implementation
DOUBLE PRECISION Double precision floating-point 8 bytes Varies by implementation

String Data Types

Data Type Description Storage
CHAR(n) Fixed-length character string n bytes
VARCHAR(n) Variable-length character string Actual length + 1 or 2 bytes
TEXT Variable unlimited length text Actual length + overhead
NCHAR(n) Fixed-length Unicode string 2×n bytes
NVARCHAR(n) Variable-length Unicode string 2×actual length + overhead

Note: CHAR always uses the specified length for storage, while VARCHAR only uses as much space as needed for the actual data plus a small overhead.

Date and Time Data Types

Data Type Description Format
DATE Date only (no time) YYYY-MM-DD
TIME Time only (no date) HH:MM:SS
DATETIME or TIMESTAMP Date and time YYYY-MM-DD HH:MM:SS
YEAR Year value YYYY

Boolean Data Type

The BOOLEAN data type can store true/false values. In some database systems, it's implemented as:

    TRUE or 1 FALSE or 0 NULL (unknown)

Binary Data Types

Data Type Description
BINARY(n) Fixed-length binary data
VARBINARY(n) Variable-length binary data
BLOB Binary Large Object for storing large binary data

Special Data Types

Data Type Description
JSON Stores JSON (JavaScript Object Notation) data
XML Stores XML data
ENUM String object with a value chosen from a list of permitted values
UUID or GUID Universally Unique Identifier

Using Data Types in Table Creation

CREATE TABLE employees (
    employee_id INT PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    birth_date DATE,
    hire_date DATE,
    salary DECIMAL(10,2),
    department VARCHAR(100),
    is_active BOOLEAN DEFAULT TRUE,
    profile_photo BLOB,
    notes TEXT
);

Data Type Conversion

Sometimes you need to convert data from one type to another. This can be done using CAST or CONVERT functions: 

-- Using CAST
SELECT CAST(123.45 AS INT);  -- Results in 123

-- Using CONVERT (in SQL Server)
SELECT CONVERT(INT, 123.45);  -- Results in 123

Best Practices:

  • Choose the smallest data type that can reliably contain all possible values
  • Use VARCHAR instead of CHAR when the column length varies considerably
  • Use INT for most integer values unless you need a larger range
  • Use DECIMAL for financial calculations where precision is important
  • Be aware of database-specific implementations of data types
