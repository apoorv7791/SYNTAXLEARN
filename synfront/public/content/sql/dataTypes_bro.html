Numeric Data Types - For All Your Counting Needs

Bro Tip: Always pick the smallest data type that fits your needs. Your database will thank you later with faster performance!

Data Type What It's For Size Range
INTEGER or INT Your go-to for whole numbers 4 bytes About -2.1 billion to 2.1 billion
SMALLINT When you're tight on space 2 bytes -32K to 32K
BIGINT For those MASSIVE numbers 8 bytes Ridiculously large (9 quintillion-ish)
DECIMAL(p,s) Money stuff - needs to be EXACT Variable Depends on precision
FLOAT Scientific calculations where precision isn't critical 4 or 8 bytes Huge range but inexact

Listen up! NEVER use FLOAT for money calculations unless you want to accidentally give away free stuff or overcharge your customers! ðŸ’¸ Always use DECIMAL for cash.

String Data Types - Text Me, Bro!

Data Type What It's For The Lowdown
CHAR(n) Fixed-length text Always takes up n bytes even if empty. Use for stuff that's always the same length (like state codes: CA, NY)
VARCHAR(n) Variable-length text The MVP of string types - only uses what it needs. Perfect for names, addresses, etc.
TEXT For your essays and novels When you need to store someone's life story or a massive JSON blob

Bro Tip: VARCHAR is your best friend 99% of the time. Only use TEXT when you're dealing with paragraphs of content.

Date and Time - Keeping Track of When Stuff Happens

Data Type What It's For Example
DATE Just the date, no time 2023-04-20
TIME Just the time, no date 13:37:00
DATETIME/TIMESTAMP The whole package - date and time 2023-04-20 13:37:00

Working with dates in SQL is like trying to text while driving - complicated and potentially disastrous. Always use the right functions to manipulate them!

Boolean - True or False, No Cap

The BOOLEAN data type is for yes/no, true/false situations:

  • TRUE (1) - "Yeah, bro!"
  • FALSE (0) - "Nah, I'm good"
  • NULL - "Umm, I don't know yet"

Perfect for columns like: is_admin, has_paid, is_verified, likes_pineapple_on_pizza

Binary Data Types - For the Heavy Stuff

Data Type What It's For
BINARY/VARBINARY Raw binary data
BLOB Binary Large Object - for storing files, images, etc.

Bro Tip: Storing files directly in your database? That's a rookie move! Better to store them on a file system or cloud storage and just keep the path in your database. Your database performance will thank you!

Special Data Types - The Cool Kids

Data Type What It's For
JSON Storing JSON data you can query - super flexible!
ENUM When you have a fixed list of options (like T-shirt sizes: S, M, L, XL)
UUID Generating unique IDs that won't collide even across different servers

Putting It All Together - Creating a Table Like a Boss

CREATE TABLE gamers (
    gamer_id INT PRIMARY KEY,                  -- Simple numeric ID
    username VARCHAR(50) NOT NULL,             -- Usernames vary in length
    email VARCHAR(100) UNIQUE NOT NULL,        -- Emails need to be unique
    password_hash CHAR(60) NOT NULL,           -- BCrypt hashes are always 60 chars
    bio TEXT,                                  -- Life stories can be long
    birth_date DATE,                           -- Just need the date, not time
    signup_date TIMESTAMP DEFAULT NOW(),       -- When they joined
    last_login TIMESTAMP,                      -- Track activity
    subscription_tier ENUM('free', 'pro', 'elite'),  -- Limited options
    account_balance DECIMAL(10,2) DEFAULT 0.00,  -- Money needs precision!
    is_verified BOOLEAN DEFAULT FALSE,         -- True/false flag
    profile_pic VARBINARY(MAX)                 -- Their awesome avatar
);

Type Conversion - Transforming Data Like a Shapeshifter

Sometimes you gotta convert data from one type to another. It's like putting on a different outfit for a different occasion: 

-- "Trust me bro, this string is actually a number"
SELECT CAST('123' AS INT);  -- Turns '123' into 123

-- "I only care about the whole number part"
SELECT CAST(123.45 AS INT);  -- Results in 123

Bro Wisdom:

  • Don't be that guy who uses VARCHAR(255) for everything just because it's the default in some tools
  • Never store dates as strings unless you enjoy debugging nightmares
  • Use DECIMAL for money, or you'll be off by fractions of cents (Office Space style)
  • When in doubt, check your database's specific documentation - every database has its own quirks
