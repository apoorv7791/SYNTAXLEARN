PHP Basic Syntax and Structure

PHP Tags

PHP code is enclosed within special start and end processing instructions (tags) that allow you to jump into and out of "PHP mode":

<?php
// PHP code goes here
?>

There are also shorthand tags, but they're not recommended as they may not be supported in all server configurations:

<?
// Shorthand tag (not recommended)
?>

Embedding PHP in HTML

One of PHP's strengths is its ability to be embedded directly in HTML:

<!DOCTYPE html>
<html>
<head>
    <title>PHP Example</title>
</head>
<body>
    <h1>Welcome to my website</h1>
    
    <?php
    $currentTime = date("H:i:s");
    echo "<p>The current time is: $currentTime</p>";
    ?>
    
    <p>This is regular HTML again.</p>
</body>
</html>

Comments

PHP supports several comment styles:

// This is a single-line comment

# This is also a single-line comment (shell-style)

/*
This is a multi-line comment
that spans multiple lines
*/

/**
 * This is a documentation comment
 * often used for documenting functions and classes
 */

Statements and Semicolons

PHP requires semicolons at the end of each statement:

<?php
echo "Hello";
echo "World";
?>

Multiple statements can be placed on the same line if each one ends with a semicolon:

<?php
echo "Hello"; echo "World";
?>

Case Sensitivity

In PHP, variable names are case-sensitive, but function names and class names are not:

<?php
$name = "John";  // Variable names are case-sensitive
$Name = "Jane";  // This is a different variable

echo $name;  // Outputs "John"
echo $Name;  // Outputs "Jane"

// Function names are NOT case-sensitive
ECHO "Hello";  // Works the same as echo
Echo "World";  // Also works
?>

Whitespace

PHP generally ignores whitespace (spaces, tabs, newlines) between statements:

<?php
echo
    "Hello"
        .
    "World";  // Outputs "HelloWorld"
?>

Echo and Print

PHP provides two basic ways to output data: echo and print:

<?php
// echo can take multiple parameters
echo "Hello", " ", "World";  // Outputs "Hello World"

// print can only take one parameter and returns a value
print "Hello World";  // Outputs "Hello World" and returns 1
?>

Variables

Variables in PHP start with a dollar sign ($) followed by the variable name:

<?php
$name = "John";
$age = 30;
$isStudent = true;

echo "Name: $name, Age: $age";  // Outputs "Name: John, Age: 30"
?>

Variable names must start with a letter or underscore, followed by any number of letters, numbers, or underscores:

<?php
$name = "John";      // Valid
$_name = "John";     // Valid
$name1 = "John";     // Valid
$1name = "John";     // Invalid (starts with a number)
$name-1 = "John";    // Invalid (contains a hyphen)
?>

String Concatenation

PHP uses the dot (.) operator to concatenate strings:

<?php
$firstName = "John";
$lastName = "Doe";
$fullName = $firstName . " " . $lastName;

echo $fullName;  // Outputs "John Doe"

// You can also use the .= operator to append to a string
$greeting = "Hello";
$greeting .= " World";
echo $greeting;  // Outputs "Hello World"
?>

String Interpolation

PHP allows variables to be directly included in double-quoted strings, but not in single-quoted strings:

<?php
$name = "John";

// Double quotes allow variable interpolation
echo "Hello, $name";  // Outputs "Hello, John"

// Single quotes do not
echo 'Hello, $name';  // Outputs "Hello, $name"

// For complex expressions, use curly braces
$fruits = ["apple", "banana", "orange"];
echo "I like {$fruits[1]}";  // Outputs "I like banana"
?>

Constants

Constants are identifiers for simple values that cannot be changed during script execution:

<?php
// Define a constant
define("PI", 3.14159);

// Use the constant
echo "The value of PI is " . PI;  // Outputs "The value of PI is 3.14159"

// Constants are global and can be used anywhere in the script
function calculateCircleArea($radius) {
    return PI * $radius * $radius;
}

echo calculateCircleArea(5);  // Outputs approximately 78.54

// Another way to define constants (PHP 5.3+)
const DATABASE_NAME = "my_database";
echo DATABASE_NAME;  // Outputs "my_database"
?>

PHP in HTML Forms

PHP is commonly used to process HTML forms:

<!DOCTYPE html>
<html>
<head>
    <title>PHP Form Example</title>
</head>
<body>
    <?php
    // Check if the form has been submitted
    if ($_SERVER["REQUEST_METHOD"] == "POST") {
        // Retrieve form data
        $name = $_POST["name"];
        $email = $_POST["email"];
        
        // Display the submitted data
        echo "<h2>Form Submission Results:</h2>";
        echo "<p>Name: $name</p>";
        echo "<p>Email: $email</p>";
    }
    ?>
    
    <h2>Contact Form</h2>
    <form method="post" action="<?php echo htmlspecialchars($_SERVER["PHP_SELF"]); ?>">
        <label for="name">Name:</label>
        <input type="text" id="name" name="name" required><br><br>
        
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required><br><br>
        
        <input type="submit" value="Submit">
    </form>
</body>
</html>

Security Tip: Always validate and sanitize user input to prevent security vulnerabilities like SQL injection and cross-site scripting (XSS).

Including Files

PHP provides several ways to include external files in your scripts:

<?php
// include: Includes and evaluates the specified file
// (produces a warning if the file is not found)
include "header.php";

// require: Same as include, but produces a fatal error if the file is not found
require "config.php";

// include_once: Includes the file only if it hasn't been included before
include_once "functions.php";

// require_once: Same as include_once, but produces a fatal error if the file is not found
require_once "database.php";
?>

PHP and JavaScript

PHP (server-side) and JavaScript (client-side) can work together:

<!DOCTYPE html>
<html>
<head>
    <title>PHP and JavaScript</title>
    <script>
    function showMessage() {
        alert("Hello from JavaScript!");
    }
    </script>
</head>
<body>
    <?php
    $message = "Hello from PHP!";
    echo "<h2>$message</h2>";
    
    // PHP can generate JavaScript code
    $jsVariable = "This was generated by PHP";
    echo "<script>";
    echo "alert('$jsVariable');";
    echo "</script>";
    ?>
    
    <button onclick="showMessage()">Click Me</button>
</body>
</html>

Best Practices

  • Always use the standard <?php ?> tags for maximum compatibility
  • End each statement with a semicolon
  • Use meaningful variable names that describe their purpose
  • Follow a consistent coding style (indentation, spacing, etc.)
  • Comment your code to explain complex logic
  • Separate PHP logic from HTML presentation when possible
  • Validate and sanitize all user inputs
  • Use error reporting during development, but disable it in production
