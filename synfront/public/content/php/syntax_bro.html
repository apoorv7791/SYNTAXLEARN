PHP Tags Kya Hote Hai? üè∑Ô∏è

Bro, PHP code ko special tags ke beech mein likhte hai, jisse PHP interpreter ko pata chalta hai ki kahan se PHP mode start karna hai aur kahan pe end:

<?php
// PHP code yahan likho
?>

Shorthand tags bhi hai, lekin inko use karna recommended nahi hai kyunki ye sab server configurations mein supported nahi hote:

<?
// Shorthand tag (recommended nahi hai)
?>

Statements aur Semicolons üìè

Bro, PHP mein har statement ke end mein semicolon (;) lagana zaroori hai:

<?php
echo "Hello";
echo "World";
?>

Multiple statements ko ek hi line mein bhi likh sakte ho, bas har ek ke baad semicolon lagao:

<?php
echo "Hello"; echo "World";
?>

Case Sensitivity üî†

Bro, PHP mein variable names case-sensitive hote hai, lekin function names aur class names case-sensitive nahi hote:

<?php
$name = "Rahul";  // Variable names case-sensitive hai
$Name = "Priya";  // Ye alag variable hai

echo $name;  // Output: "Rahul"
echo $Name;  // Output: "Priya"

// Function names case-sensitive NAHI hai
ECHO "Hello";  // Bilkul echo ki tarah kaam karega
Echo "World";  // Ye bhi kaam karega
?>

Whitespace üåå

Bro, PHP generally whitespace (spaces, tabs, newlines) ko statements ke beech ignore kar deta hai:

<?php
echo
    "Hello"
        .
    "World";  // Output: "HelloWorld"
?>

Echo aur Print üì¢

Bro, PHP mein data output karne ke do basic tarike hai: echo aur print:

<?php
// echo multiple parameters le sakta hai
echo "Hello", " ", "World";  // Output: "Hello World"

// print sirf ek parameter le sakta hai aur value return karta hai
print "Hello World";  // Output: "Hello World" aur 1 return karega
?>

Variables Kaise Banate Hai? üì¶

Bro, PHP mein variables dollar sign ($) se start hote hai, uske baad variable name aata hai:

<?php
$name = "Rahul";
$age = 25;
$isStudent = true;

echo "Name: $name, Age: $age";  // Output: "Name: Rahul, Age: 25"
?>

Variable names letter ya underscore se start hone chahiye, uske baad koi bhi number of letters, numbers, ya underscores ho sakte hai:

<?php
$name = "Rahul";     // Valid
$_name = "Rahul";    // Valid
$name1 = "Rahul";    // Valid
$1name = "Rahul";    // Invalid (number se start ho raha hai)
$name-1 = "Rahul";   // Invalid (hyphen contain kar raha hai)
?>

String Concatenation üîó

Bro, PHP mein strings ko jodne ke liye dot (.) operator use hota hai:

<?php
$firstName = "Rahul";
$lastName = "Sharma";
$fullName = $firstName . " " . $lastName;

echo $fullName;  // Output: "Rahul Sharma"

// .= operator se existing string mein add kar sakte ho
$greeting = "Hello";
$greeting .= " Bhai";
echo $greeting;  // Output: "Hello Bhai"
?>

String Interpolation üîÑ

Bro, PHP mein double quotes ke andar variables direct include ho sakte hai, lekin single quotes mein nahi:

<?php
$name = "Rahul";

// Double quotes variable interpolation allow karte hai
echo "Hello, $name";  // Output: "Hello, Rahul"

// Single quotes nahi karte
echo 'Hello, $name';  // Output: "Hello, $name"

// Complex expressions ke liye curly braces use karo
$fruits = ["apple", "banana", "orange"];
echo "Mujhe {$fruits[1]} pasand hai";  // Output: "Mujhe banana pasand hai"
?>

Constants Kya Hote Hai? üóø

Bro, constants aise identifiers hai jo simple values store karte hai aur script execution ke dauraan change nahi ho sakte:

<?php
// Constant define karo
define("PI", 3.14159);

// Constant use karo
echo "PI ki value hai " . PI;  // Output: "PI ki value hai 3.14159"

// Constants global hote hai aur script ke kahin bhi use kar sakte hai
function calculateCircleArea($radius) {
    return PI * $radius * $radius;
}

echo calculateCircleArea(5);  // Output: approximately 78.54

// Constants define karne ka ek aur tarika (PHP 5.3+)
const DATABASE_NAME = "my_database";
echo DATABASE_NAME;  // Output: "my_database"
?>

PHP in HTML Forms üìù

Bro, PHP aksar HTML forms ko process karne ke liye use hota hai:

<!DOCTYPE html>
<html>
<head>
    <title>PHP Form Example</title>
</head>
<body>
    <?php
    // Check karo ki form submit hua hai ya nahi
    if ($_SERVER["REQUEST_METHOD"] == "POST") {
        // Form data retrieve karo
        $name = $_POST["name"];
        $email = $_POST["email"];
        
        // Submitted data display karo
        echo "<h2>Form Submission Results:</h2>";
        echo "<p>Name: $name</p>";
        echo "<p>Email: $email</p>";
    }
    ?>
    
    <h2>Contact Form</h2>
    <form method="post" action="<?php echo htmlspecialchars($_SERVER["PHP_SELF"]); ?>">
        <label for="name">Name:</label>
        <input type="text" id="name" name="name" required><br><br>
        
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required><br><br>
        
        <input type="submit" value="Submit">
    </form>
</body>
</html>

Security Tip: Bro, hamesha user input ko validate aur sanitize karo! Warna SQL injection aur cross-site scripting (XSS) jaise security vulnerabilities ho sakte hai.

Files Kaise Include Karte Hai? üìÇ

Bro, PHP mein external files ko include karne ke kai tarike hai:

<?php
// include: Specified file ko include aur evaluate karta hai
// (agar file nahi mili to warning produce karta hai)
include "header.php";

// require: include ki tarah hai, lekin agar file nahi mili to fatal error produce karta hai
require "config.php";

// include_once: File ko sirf tab include karta hai jab wo pehle include nahi hui ho
include_once "functions.php";

// require_once: include_once ki tarah hai, lekin agar file nahi mili to fatal error produce karta hai
require_once "database.php";
?>

Best Practices üíØ

  • Hamesha standard <?php ?> tags use karo maximum compatibility ke liye
  • Har statement ke end mein semicolon lagao
  • Meaningful variable names use karo jo unke purpose ko describe kare
  • Consistent coding style follow karo (indentation, spacing, etc.)
  • Complex logic ko explain karne ke liye code ko comment karo
  • Jahan possible ho, PHP logic ko HTML presentation se separate rakho
  • Sab user inputs ko validate aur sanitize karo
  • Development ke time error reporting enable rakho, lekin production mein disable karo
